## The 8-Agent Team

| Phase | Agent | Role |
|---|---|---|
| 1 | 业务规划师 / Business Planner | Strategic frame, scenario level (L1–L5), 3-phase roadmap |
| 2 | 产品经理 / Product Manager | Lean AI scenario card, ROI model, KPI dashboard, stop conditions |
| 3 | 业务分析师 / Business Analyst | User stories, As-Is→To-Be flows, acceptance criteria |
| 4 | 技术架构师 / Technical Architect | ADRs, API contracts, Clean Core design, governance |
| 5 | 前端开发 / Frontend | UI implementation per API contracts |
| 5 | 后端开发 / Backend | API implementation per contracts |
| 5 | 数据集成 / Data Integration | Schema, knowledge base, data feedback loop |
| 6 | 合规项目管理 / Compliance PM | Four-loop check, conflict detection, DoD |

## Lean AI Four Closed Loops

Every task is checked against:
- **Value Loop** — measurable business outcomes defined?
- **Data Loop** — AI outputs feed back into future improvement?
- **Model Loop** — system supports future model upgrades?
- **Operations Loop** — KPIs and monitoring in place before launch?

## Methodology

Based on **精益AI方法论** (Lean AI Methodology) by 史凯 Kai Shi:

> "AI transformation is not model procurement — it is lean reconstruction of processes, data, organization, technology, and operations, with business scenarios as the core."

**10 Core Principles:**
1. Scenario First — find the scenario before selecting the model
2. Value-Driven — no business metric = no scale investment
3. Small Steps, Fast Cycles — POC → MVP → scale
4. Data Before Code — high-quality AI needs high-quality data
5. Knowledge Compounds — the moat is business knowledge, not models
6. Human-AI Collaboration — AI restructures division of labor
7. Controlled Execution — governance increases with business proximity
8. Continuous Operations — launch is the beginning, not the end
9. Platform Grows from Scenarios — don't build platforms first
10. Organizations Must Evolve — no org change = no real AI transformation
