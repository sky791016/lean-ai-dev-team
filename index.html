Why Claude Code Skills

AI Dev Teams Need a Brain, Not Just Tools

Most AI coding tools generate code. Skills generate coordinated systems — where every agent knows its role, reads existing code, and hands off to the next stage.

🎯

Persistent Methodology

Skills are reusable playbooks baked into Claude Code. Once installed, every invocation follows the same Lean AI sequence — no prompting required.

🔄

Sequential + Parallel Execution

Strategy agents run first, then developers run in parallel. Skills orchestrate this automatically — you get 6 phases of coordinated output.

📊

Business Value Embedded

Every phase produces ROI measurements, scenario cards, and success metrics — not just code. Business and engineering outputs are unified in one workflow.

🏗️

Read Before You Write

Every agent is instructed to read existing code before generating anything. No phantom modules, no mismatched APIs — the team works with your actual codebase.

⚖️

Four Closed-Loop Guarantee

Value loop, data loop, model loop, and operations loop are checked at the end of every task. Nothing ships without all four confirmed.

🛡️

Compliance Built In

The Compliance PM agent runs last on every task — reviewing data privacy, access control, audit logging, and AI governance before DoD is signed.

The 8-Agent Team

Strategy to Code, Without Gaps

Each agent owns a distinct domain. Earlier agents set constraints that later agents must honor — especially API contracts between Architect → Frontend → Backend.

Phase 1 · Strategy
业务规划师 / Business Planner
Sets the strategic frame before any analysis begins. Defines the business case, stakeholder map, scenario level (L1–L5), and 3-stage roadmap.
Business Case L1–L5 Scenario Level 3-Phase Roadmap
Phase 2 · Product
产品经理 / Product Manager
Owns commercialization and ROI. Produces the Lean AI scenario card, quantified ROI model, success KPI dashboard, and go/no-go criteria.
Scenario Card ROI Model KPI Dashboard Stop Conditions
Phase 3 · Analysis
业务分析师 / Business Analyst
Translates strategy into engineering specs. User stories, As-Is → To-Be flows, Given/When/Then acceptance criteria, human-AI handoff design.
User Stories As-Is / To-Be Acceptance Criteria Human-AI Split
Phase 4 · Architecture
技术架构师 / Technical Architect
Designs the system before any code is written. ADRs, Clean Core + Cognitive Sidecar design, full API contracts, governance framework, data flows.
ADRs API Contracts Clean Core Design Governance Plan
Phase 5 · Build (Parallel)
前端开发 / Frontend
Implements UI following the architect's API contracts. Reads existing components first, builds human-AI collaboration interfaces, exposes operational dashboards.
Component Changes API Usage Map Ops Dashboard UI
Phase 5 · Build (Parallel)
后端开发 / Backend
Implements APIs per the architect's contracts. Follows existing service patterns, adds @Transactional for multi-table writes, logs all controlled actions.
Endpoints List Service Logic Audit Logs
Phase 5 · Build (Parallel)
数据集成 / Data Integration
Handles schema, migrations, knowledge base design, and external APIs. Crucially: designs the data feedback loop so AI outputs improve future models.
Migration SQL Data Flow Feedback Loop Knowledge Base
Phase 6 · Closure
合规项目管理 / Compliance PM
Consolidates all outputs, runs the four-loop check, detects interface conflicts, produces ordered execution checklist and Definition of Done.
4-Loop Check Conflict Report Execution Checklist DoD

Lean AI Framework

Built on 一心·两翼·三层·四闭环

Every agent prompt is aligned to Kai Shi's Lean AI Framework — the same methodology used for enterprise AI transformation at scale.

Value Loop

🗄️

Execution Flow

How the 6 phases connect

Phase
1

业务规划师 — Strategic Frame

Sets scenario level (L1–L5), value type, stakeholder map, and 3-phase roadmap. Nothing begins until "why" is answered.

Phase
2

产品经理 — Scenario Card & ROI

Produces the standard Lean AI scenario card, quantified ROI model, KPI dashboard template, and stop conditions.

Phase
3

业务分析师 — Engineering Specs

User stories, process maps (As-Is → To-Be), acceptance criteria, human-AI handoff nodes, and data requirements.

Phase
4

技术架构师 — System Blueprint

ADRs, Clean Core + Cognitive Sidecar design, full API contracts, governance framework. Frontend and backend must honor this.

Phase
5

前端 + 后端 + 数据集成 PARALLEL

Three agents build simultaneously. All three work from the architect's API contracts and read existing code before writing.

Phase
6

合规项目管理 — Four-Loop Closure

Consolidates all outputs. Runs value, data, model, and ops loop checks. Flags interface conflicts. Signs off on DoD.

Get Started

Install in 3 Steps

Requires Claude Code (claude.ai/code). Works with any codebase.

  1. Clone the skill

    # Clone into your Claude skills directory
    git clone https://github.com/yourusername/lean-ai-dev-team \
      ~/.claude/skills/dev-team

  2. Verify installation

    Open Claude Code in your project. The skill auto-loads from ~/.claude/skills/. No configuration needed.

  3. Invoke the team

    # In any Claude Code session:
    /dev-team Add a real-time notification system for user events

    # Or invoke directly:
    "Run dev-team: implement user dashboard with ROI tracking"

When to Use Each Mode

Full 8-agent team — new features, AI-native products, cross-layer changes, anything touching strategy + code.

Lightweight mode — bug fixes start at Phase 3 (Analyst), pure UI work starts at Phase 5. Skip agents you don't need.
